# Stock Forecast

Веб-приложение для прогнозирования цен на акции с помощью методов машинного обучения. Позволяет выбрать финансовый инструмент, загрузить его исторические котировки и получить прогноз на заданный горизонт одной из двух моделей — Random Forest или LSTM — либо сравнить прогнозы обеих моделей на одном графике.

![Python](https://img.shields.io/badge/python-3.10+-blue.svg)

## Возможности

- Загрузка исторических котировок с Yahoo Finance через библиотеку yfinance
- Кэширование запросов в PostgreSQL для снижения нагрузки на внешний API
- Расчёт нескольких десятков технических индикаторов (SMA, EMA, MACD, RSI, полосы Боллинджера, ATR, OBV, MOM, ROC, Williams %R, CCI, ADX), лаговых значений и оценок волатильности
- Прогноз цены на 1–30 торговых дней вперёд
- Две модели на выбор: Random Forest (scikit-learn) и LSTM (TensorFlow/Keras)
- Режим сравнения моделей на одном графике с автоматическим определением лучшей по метрикам RMSE, MAE и MAPE
- Интерактивные графики Plotly: свечной, прогнозный, сравнительный, тестовый
- Хранение обученных моделей и метаданных (гиперпараметры, метрики качества, время обучения) в PostgreSQL

## Требования

- Python 3.10 или выше
- PostgreSQL 15+ (локально или удалённый сервер)
- TA-Lib — нативная библиотека на языке C, устанавливается отдельно от Python-обёртки

### Установка TA-Lib

**Windows:** скачать whl-файл с [unofficial Windows binaries](https://github.com/cgohlke/talib-build/releases) под свою версию Python и архитектуру, затем:
```bash
pip install TA_Lib-*.whl
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt install ta-lib
pip install TA-Lib
```

## Установка и запуск

```bash

cd stock_forecast
```

Параметры подключения задаются через переменные окружения. Если их не задать, используются значения по умолчанию из `config.py` (localhost, postgres/postgres).

## Использование

На боковой панели выбирается режим работы:

- **Историческая динамика** — свечной график с объёмами торгов и статистикой за выбранный период
- **Прогноз** — прогноз выбранной моделью (Random Forest или LSTM) на горизонт от 1 до 30 торговых дней с метриками качества и таблицей прогнозных значений
- **Сравнение моделей** — прогнозы обеих моделей на одном графике, сравнительная таблица и автоматическое определение лучшей модели по RMSE, MAE и MAPE

В конфигурации `config.py` задан список из 14 тикеров (AAPL, MSFT, GOOGL, AMZN, TSLA, NVDA, JPM, V, JNJ, WMT, PG, DIS, NFLX, INTC). Добавить свой инструмент можно, дописав его в словарь `AVAILABLE_STOCKS`.

### Модули

**Модуль работы с данными (`modules/data_module.py`)**

- `DataLoader` — загрузка котировок. Сначала проверяет кэш в PostgreSQL, при промахе обращается к yfinance и сохраняет результат в базу.
- `DataPreprocessor` — очистка данных: удаление дубликатов по дате, сортировка по времени, отбрасывание строк с отсутствующими или нулевыми ценами закрытия.
- `FeatureEngineer` — формирование матрицы признаков: расчёт технических индикаторов через TA-Lib, лаговых значений, доходностей, оценок волатильности и календарных признаков.
- `CacheManager` — управление кэшем запросов. Хеширует комбинацию «тикер + период» алгоритмом SHA-256 и проверяет наличие записи в таблице `request_cache`.

**Модуль моделей (`modules/models/model_module.py`)**

- `BaseModel` — абстрактный базовый класс с пятью обязательными методами: `train`, `predict`, `save`, `load`, `get_params`. Любая новая модель наследуется от него и автоматически становится совместимой с `ModelTrainer`.
- `RandomForestModel` — реализация на scikit-learn. Содержит eval-модель для оценки качества (обучается на дневных доходностях) и словарь из 30 моделей для Direct Multi-Step Forecasting (каждая обучена на кумулятивных доходностях своего горизонта).
- `LSTMModel` — трёхслойная рекуррентная сеть на Keras (64 → 32 → 16 нейронов с Dropout). Обучается предсказывать дневную доходность, многошаговый прогноз строит рекурсивно.
- `ModelTrainer` — полный цикл обучения: разбиение выборки 80/20 с сохранением хронологического порядка, тренировка, оценка по RMSE/MAE/MAPE, сохранение модели на диск и метаданных в БД.
- `Predictor` — загружает последнюю обученную модель для инструмента и генерирует прогнозы.

**Модуль визуализации (`modules/visualization/`)**

- `ChartGenerator` — методы построения графиков через Plotly: свечной график с объёмами, график истории и прогноза одной модели, сравнительный график двух моделей, график «факт vs. прогноз» на тестовой выборке.
- `TableFormatter` — форматирование таблиц прогнозов, сравнительных таблиц и сводки метрик.
- `MetricsCalculator` — расчёт RMSE, MAE и MAPE с корректной обработкой нулевых значений при делении.

**База данных (`database/db_manager.py`)**

Класс `DBManager` инкапсулирует всё взаимодействие с PostgreSQL. Схема содержит четыре таблицы:

- `stocks` — справочник инструментов (тикер + название)
- `historical_data` — исторические котировки (OHLCV), с уникальным индексом на пару `(ticker, date)` для идемпотентной загрузки
- `model_metadata` — обученные модели: тип, путь к файлу, метрики на тесте, гиперпараметры в JSONB
- `request_cache` — SHA-256-хеши запросов для пропуска повторных обращений к внешнему API

Все связи реализованы через внешние ключи с каскадным удалением.

## Особенности реализации

**Обучение на доходностях, а не на абсолютных ценах.** Обе модели предсказывают относительное изменение цены, а не саму цену. Это обеспечивает масштабную инвариантность (модель, обученная на акциях по 150 USD, корректно работает на акциях по 3000 USD) и делает прогноз направленно-ориентированным.

**Random Forest: Direct Multi-Step Forecasting.** Для каждого горизонта h от 1 до 30 обучается отдельный `RandomForestRegressor`, предсказывающий кумулятивную доходность за h дней. Подход избегает накопления ошибок рекурсивного прогноза.

**LSTM: рекурсивный многошаговый прогноз.** На каждом шаге нейросеть возвращает предсказанную дневную доходность, окно сдвигается на один шаг, процесс повторяется до достижения заданного горизонта. Доходность на каждом шаге ограничивается диапазоном [−5%, +5%] для исключения нереалистичных выбросов.

**Метрики — однодневные.** RMSE, MAE и MAPE рассчитываются для прогноза на один шаг вперёд. Это позволяет корректно сравнивать обе модели в одинаковых условиях и сохраняет интерпретируемость ошибки, которая на длинных горизонтах рекурсивного прогноза размывается шумом.

**Сохранение LSTM в один файл.** Веса нейросети, параметры нормализации (mean и scale объекта `StandardScaler`), размер окна и гиперпараметры записываются в единый HDF5-файл через `h5py`. Для каждого инструмента и типа модели существует ровно один файл — не требуется синхронизировать несколько.

## Ограничения

- Источник данных (yfinance) не имеет официальных гарантий доступности и скорости обновления.
- Модели тренируются на исторических паттернах и не учитывают новостной фон, корпоративные события, макроэкономические шоки.
- На длинных горизонтах прогноза (20–30 дней) точность падает из-за накопления ошибок.
- Система предназначена для демонстрационных и образовательных целей и не является инвестиционной рекомендацией.
